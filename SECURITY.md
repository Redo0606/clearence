# Security Policy

## Supported Versions

We release patches for security vulnerabilities for the latest release. Older versions are not actively supported.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately rather than opening a public issue.

**How to report:**

1. **GitLab (preferred):** Create a confidential issue in the [project's issue tracker](https://gitlab.com/YOUR_ORG/clearence/-/issues) and mark it as confidential, or contact the maintainers directly via the project's contact information.

2. **Email:** If you have maintainer contact details, you may report via email.

**What to include:**

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

**Response:**

- We aim to acknowledge reports within 48 hours
- We will keep you informed of progress and resolution
- We credit reporters in security advisories (unless you prefer to remain anonymous)

**Please do not** disclose the vulnerability publicly until we have had a chance to address it.
